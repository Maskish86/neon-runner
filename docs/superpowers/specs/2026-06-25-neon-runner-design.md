# NEON RUNNER — Design Spec

**Date:** 2026-06-25  
**Status:** Approved

---

## Overview

ブラウザベースの3レーン型エンドレスランナー。サイバーパンク巨大都市の屋上を舞台に、プレイヤーは追跡するセキュリティドローンから逃げながら障害物を回避し、データシャードを収集する。

**Tech stack:** Vite + Vanilla JS + Three.js。外部アセットなし（プリミティブ・マテリアルのみ）。

---

## Architecture

### Rendering

- **フルThree.js**アプローチ。すべての3Dオブジェクト（背景ビル含む）を単一シーンで管理。
- HUDのみDOM/CSS overlay。
- `PerspectiveCamera` をプレイヤー後方上方に固定。プレイヤーは画面中央下部に配置。
- 背景ビル群: `InstancedMesh` で管理。カメラ速度の0.3倍でZ移動（パララックス）。
- 遠景スカイライン: 固定の半透明プレーン（ネオングラデーション）。
- `FogExp2` でデプスフォグ（カラー: ネオンパープル）。

### Object Pooling

障害物・シャード・パーティクルはすべて固定数プールから再利用。使用済みオブジェクトは位置をリセットして再スポーン。`dispose()` はセッション終了時のみ。

### Game State Machine

```
TITLE → PLAYING → GAME_OVER → TITLE
                ↑         |
                └─RESTART─┘
```

`main.js` がステートを保持し、各モジュールへ `update(delta, state)` を委譲。

## Player

### Shape

ボックス人型（頭・胴体・腕・脚、計6メッシュを `THREE.Group` 化）。エミッシブシアン/マゼンタのアウトライン。キャラクター色はスタート画面で3種選択可（シアン/マゼンタ/ゴールド）。

### Lane System

| レーン | X座標 |
|--------|-------|
| 左     | -2.5  |
| 中央   | 0.0   |
| 右     | +2.5  |

### State Machine

```
IDLE → RUNNING → [LANE_SWITCH | JUMPING | SLIDING] → RUNNING
```

- **レーン切り替え:** `lerp` でスムーズ移動（約0.15秒）。切り替え中も再入力1つキューイング可。
- **ジャンプ:** 放物線（初速 + 重力定数）。滞空中再ジャンプ不可。
- **スライド:** 0.6秒間、AABBの高さを半減。胴体・頭メッシュを `scale.y` 縮小で表現。

### HP & Invincibility

- 3ライフ（ハートアイコン×3 on HUD）。
- 被弾時0.8秒の無敵時間 + プレイヤーメッシュ点滅。

### Speed Scaling

```js
speed = BASE_SPEED + distance * ACCEL_FACTOR  // MAX_SPEEDでキャップ
```

## Obstacles

### Types (`obstacle-types.js`)

| 種別           | 回避方法              | 形状                             |
|----------------|-----------------------|----------------------------------|
| ホログラム看板 | ジャンプ              | 薄いボックス、エミッシブブルー   |
| ネオンパイプ   | スライド              | 横長シリンダー、シアン発光       |
| 屋上の隙間     | ジャンプ              | 地面テクスチャ欠落＋ダークボイド |
| レーザーゲート | ジャンプorスライド    | 赤ライン、点滅周期あり           |
| パトロールボット | レーン変更           | ボックス小型人型、横移動         |

### Spawning (`obstacles.js`)

- Zカメラ前方60ユニットに生成。Z=-10で回収してプールへ返却。
- 難易度: `distance` に応じてスポーン間隔を短縮、密度増加。
- 同一レーン連続スポーン制限（プレイアビリティ確保）。

## Collectibles

### Data Shards

- 形状: 八面体ジオメトリ、シアン/マゼンタ交互。Y軸回転アニメ。
- レーンに沿って配置。スコア +10 per shard。

### Power-ups

| 種別          | 色   | 効果                                          | 持続  |
|---------------|------|-----------------------------------------------|-------|
| シールド      | 青   | ダメージ1回吸収                               | 1hit  |
| マグネット    | 黄   | 隣接レーンのシャードをlerp吸引               | 8s    |
| オーバードライブ | 白 | 無敵 + スピードバースト + ネオントレイル強化 | 5s    |
| ホバーブーツ  | 緑   | 低高度飛行、障害物上空通過                   | 6s    |

マグネット: 毎フレーム、隣接レーンのアクティブシャードをプレイヤーX座標へ `lerp` 移動。

---

## Collision

- AABB（軸並行境界ボックス）ベース。`collision.js` が全アクティブ障害物・収集アイテムと判定。
- スライド中はプレイヤーAABB高さを半減して判定。
- ホバーブーツ中は地上障害物との判定を無効化。

---

## Drone System (`drone.js`)

### Proximity Model

`proximity`: 0.0〜1.0 の浮動小数値。

| イベント             | 変化量 |
|----------------------|--------|
| 障害物衝突           | +0.3   |
| 速度低下（被弾後）   | +time  |
| 回避成功             | -0.1   |
| オーバードライブ使用 | -0.4   |

`proximity >= 1.0` → ゲームオーバートリガー。

### Visual Feedback

- 画面端の赤グロー（DOM overlay の `box-shadow`）の強度 = `proximity`。
- `proximity > 0.6`: HUD中央下に警告テキスト表示。
- サウンド（`audio.js`実装時）: 接近に応じて警告音の音量・周波数を増加。

### Game Over Sequence

`proximity >= 1.0` 時のみ実際の3Dドローンメッシュを生成（ボックス群構成）。カメラ後方から急降下アニメーション → ゲームオーバー画面トリガー。

## Game Flow

### Start Screen
- タイトル「NEON RUNNER」（CSSグロー）
- キャラクター色選択: シアン / マゼンタ / ゴールド
- 操作説明（キーボード + スワイプ）
- 「PRESS SPACE / TAP TO START」

### Game Over Screen
- 走行距離・収集シャード数・ハイスコア
- Restart ボタン

## Input

| アクション | キーボード              | モバイル     |
|-----------|------------------------|-------------|
| 左        | ← / A                 | 左スワイプ  |
| 右        | → / D                 | 右スワイプ  |
| ジャンプ  | ↑ / W / Space         | 上スワイプ  |
| スライド  | ↓ / S                 | 下スワイプ  |

デスクトップ・モバイル共通の入力レイヤーを `main.js` で管理。ロジック重複なし。

## Visual Style

- 背景: ネオンピンク/シアン/パープルグラデーションのスカイライン
- エミッシブマテリアルで障害物・収集アイテムのグロー表現
- `FogExp2` でデプスフォグ
- オプション: 雨/霧エフェクト（パフォーマンス切り替え可能）

---

## Performance Targets

- 60fps維持。
- `InstancedMesh` で背景ビル描画コスト削減。
- オブジェクトプールで GC プレッシャー最小化。
- `requestAnimationFrame` ベースのゲームループ（Three.jsの `renderer.setAnimationLoop`）。

---

## Build Order

1. Viteプロジェクト足場 + 基本Three.jsシーン（カメラ・地面・照明）
2. プレイヤー移動・レーン切り替え（コアフィール確立）
3. 障害物スポーン + AABB当たり判定 + コアループ（走る・避ける・死ぬ・リスタート）
4. 収集アイテム + パワーアップ
5. ドローン接近度システム + 難易度スケーリング
6. ブラッシュアップ（パーティクル・グロー・パララックス・HUDスタイリング）
7. オーディオ（別途相談）
